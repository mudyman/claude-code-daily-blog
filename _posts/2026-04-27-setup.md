# 每日博学 Claude Code：从零开始，打造你的 AI 结对编程环境

作为一个每天在终端里摸爬滚打的开发者，你是否厌倦了在编辑器和浏览器之间反复横跳，只为了查 API、写正则、或者理解一段晦涩的遗留代码？Claude Code 的出现，让 AI 真正走进了你的工作流——它不是一个悬浮的聊天窗，而是能直接读写代码、执行命令的结对编程伙伴。但这一切的前提是：你得先把它请进你的终端。

今天，我们就来聊聊如何从零安装、配置并验证 Claude Code，为你的 AI 驱动开发之旅铺平道路。

---

## 关键概念：系统要求与环境依赖

在动手之前，先确认你的环境是否达标。Claude Code 对系统的要求并不苛刻，但有几个硬性条件：

- **操作系统**：macOS 13.0+、Windows 10 1809+（或 Windows Server 2019+）、Ubuntu 20.04+、Debian 10+、Alpine Linux 3.19+
- **硬件**：4 GB+ 内存，x64 或 ARM64 处理器
- **网络**：必须联网
- **Shell**：Bash、Zsh、PowerShell 或 CMD
- **账号**：需要 Pro、Max、Team、Enterprise 或 Console 账号（免费版 Claude.ai 无法使用 Claude Code）

**特别提醒**：如果你使用原生 Windows 环境，必须提前安装 [Git for Windows](https://git-scm.com/downloads/win)，因为 Claude Code 底层依赖 Git Bash 来执行命令。如果你在 WSL 中运行，则不需要。

---

## 实操演示：三步搞定安装与验证

### 1. 一键安装

根据你的操作系统，选择对应的命令。官方提供了原生的安装脚本，这也是最推荐的方式。

**macOS, Linux, WSL:**

```bash
curl -fsSL https://claude.ai/install.sh | bash
```

**Windows PowerShell:**

```powershell
irm https://claude.ai/install.ps1 | iex
```

**Windows CMD:**

```batch
curl -fsSL https://claude.ai/install.cmd -o install.cmd && install.cmd && del install.cmd
```



![Cli Install Command](assets/images/screenshots/cli-install-command.png)



如果你是包管理器的忠实拥趸，也可以通过 Homebrew (macOS) 或 WinGet (Windows) 安装：

```bash
brew install --cask claude-code
```

```powershell
winget install Anthropic.ClaudeCode
```

**避坑指南**：如果你在 Windows 上运行 CMD 命令时遇到 `The token '&&' is not a valid statement separator`，说明你当前在 PowerShell 环境中；反之，如果提示 `'irm' is not recognized`，则说明你在 CMD 中。看清终端提示符的 `PS` 前缀能帮你少走弯路。

### 2. 验证安装

安装完成后，在终端输入：

```bash
claude --version
```



![Version Check Output](assets/images/screenshots/version-check-output.png)



如果想进行更全面的健康检查（包括依赖和配置诊断），运行：

```bash
claude doctor
```

这步非常实用，如果后续遇到奇怪的问题，先用 `claude doctor` 体检一下准没错。

### 3. 登录认证

在项目目录下启动 Claude Code：

```bash
claude
```

首次运行会引导你通过浏览器完成 OAuth 认证。认证成功后，你就可以正式与 AI 对话了。

## 进阶技巧：版本控制与深度定制

对于追求稳定的企业团队或对版本有严格要求的开发者，Claude Code 提供了精细的版本管理机制。

### 选择发布通道

默认情况下，原生安装的 Claude Code 会在后台自动更新（`latest` 通道）。如果你希望更稳定，可以切换到 `stable` 通道，它通常比最新版晚一周左右，跳过了存在重大退步的版本。

你可以通过 `/config` 设置，或者直接在 `settings.json` 中配置：

```json
{
  "autoUpdatesChannel": "stable"
}
```

### 锁定最低版本

如果你当前在 `latest` 通道，且不想因为切换到 `stable` 被意外降级，可以通过 `minimumVersion` 锁定下限：

```json
{
  "autoUpdatesChannel": "stable",
  "minimumVersion": "2.1.100"
}
```

### 安装特定版本

在排查兼容性问题时，你可能需要安装特定版本。原生安装脚本支持传入版本号或通道参数：

**安装 stable 版本:**

```bash
curl -fsSL https://claude.ai/install.sh | bash -s stable
```

**安装特定版本 (如 2.1.89):**

```bash
curl -fsSL https://claude.ai/install.sh | bash -s 2.1.89
```

### 彻底卸载

如果需要重装或彻底清理，除了移除二进制文件，别忘了清理配置文件（注意：这会删除你的会话历史和设置）：

**移除程序 (macOS/Linux/WSL):**

```bash
rm -f ~/.local/bin/claude
rm -rf ~/.local/share/claude
```

**清理配置与缓存:**

```bash
# Remove user settings and state
rm -rf ~/.claude
rm ~/.claude.json

# Remove project-specific settings (run from your project directory)
rm -rf .claude
rm -f .mcp.json
```
