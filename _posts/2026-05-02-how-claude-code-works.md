# 揭秘 Claude Code 的核心引擎：智能体循环如何重塑你的开发流

你有没有想过，为什么传统的 AI 编程助手总是让你觉得“差一口气”？它们只能看着你当前打开的文件，给出片段式的建议，一旦涉及跨文件重构或跑测试，就瞬间变成了“旁观者”。

Claude Code 的出现打破了这种局限。它不仅是一个代码补全工具，更是一个能在你的命令行里独立思考、执行和验证的**智能体**。今天，我们就来拆解 Claude Code 的底层运作机制，看看它是如何真正接管你的开发工作流的。

## 核心机制：智能体循环

当你给 Claude Code 下达一个任务时，它并不是简单地一次性生成答案，而是进入一个持续运转的**智能体循环**。这个循环包含三个相互交织的阶段：**收集上下文**、**采取行动** 和 **验证结果**。



![Agentic Loop Diagram](assets/images/screenshots/agentic-loop-diagram.png)



这个循环的强大之处在于它的**自适应能力**：
- 如果你只是问个代码库的问题，它可能只需收集上下文。
- 如果是修 Bug，它会在三个阶段间反复横跳：跑测试、看报错、找源码、改代码、再跑测试。
- 如果是大型重构，它会进行极其严密的验证。

更重要的是，**你始终是这个循环的一部分**。你可以随时打断它，提供新线索，或者让它换条路走。它自主运行，但绝不失控。

支撑这个循环的，是两大支柱：负责推理的**模型**和负责行动的**工具**。

### 模型：大脑

Claude Code 使用 Claude 模型来理解代码和推理任务。面对复杂任务，它会将工作拆解为步骤，逐步执行并根据反馈调整。你可以根据需求切换模型：Sonnet 处理大多数日常编码游刃有余，而 Opus 则在复杂架构决策上提供更强的推理能力。你可以用 `/model` 随时切换。

### 工具：双手

没有工具，Claude 只能跟你“纸上谈兵”。有了工具，它才能真正操作你的环境。每次工具执行后的返回信息，都会喂给模型，决定下一步怎么走。

它内置了五大类工具：

| 类别 | 能力 |
| --- | --- |
| **文件操作** | 读取、编辑、创建、重命名文件 |
| **搜索** | 按模式查找文件、正则搜索内容、探索代码库 |
| **执行** | 运行 Shell 命令、启动服务、跑测试、使用 Git |
| **网络** | 搜索网页、抓取文档、查询报错信息 |
| **代码智能** | 查看类型报错、跳转定义、查找引用（需插件支持） |

举个真实场景，当你说“修复失败的测试”时，Claude 会：1. 跑测试套件看报错 -> 2. 读取错误输出 -> 3. 搜索相关源文件 -> 4. 读取文件理解逻辑 -> 5. 修改代码 -> 6. 再次跑测试验证。这就是智能体循环的完美展现。

## 它能看到什么？

当你在项目目录下运行 `claude` 时，它就拥有了该项目的全局视野：

- **你的项目**：当前及子目录下的所有文件（访问外部文件需你授权）。
- **你的终端**：你能跑的命令它都能跑。
- **Git 状态**：当前分支、未提交的更改、近期提交历史。
- **CLAUDE.md 与自动记忆**：跨会话持久化的项目规范和偏好设置。

这种全局视野让它与传统的“单文件内联助手”有了本质区别。修一个认证 Bug 时，它能自己搜文件、看上下文、跨文件协调修改、跑测试、甚至帮你提交代码。

## 会话与上下文管理

### 会话的延续与分叉

Claude Code 会将对话保存在本地的 JSONL 文件中。你可以用 `claude --continue` 恢复上一次的对话，也可以用 `claude --resume` 选择历史会话。

如果你想基于当前进度尝试另一种方案，又不想破坏原会话，可以使用分叉功能：

```bash
claude --continue --fork-session
```

这会保留之前的对话历史，但生成一个新的会话 ID，原会话不受影响。**强烈建议**：不要在多个终端恢复同一个会话，这会导致消息交错混乱；如果需要并行工作，请使用 `--fork-session` 给每个终端独立的会话。

### 上下文窗口的生存法则

上下文窗口是 Claude 的“工作记忆”，装着对话历史、文件内容、命令输出等。随着工作推进，上下文会被填满。虽然 Claude 会自动压缩旧内容，但早期的细节仍可能丢失。

**生存法则**：不要把重要规则只留在对话里，一定要写进 `CLAUDE.md`。你可以运行 `/context` 查看空间占用情况。如果需要手动压缩并保留重点，可以使用：

```bash
/compact focus on the API changes
```

此外，**子代理** 拥有完全独立的上下文，不会撑爆主会话；**技能** 则是按需加载，只在用到时才占用上下文。

## 安全防线：检查点与权限

把控制权交给 AI，安全怎么保障？Claude Code 有双重防线：

1. **检查点**：每次编辑文件前，Claude 都会打快照。搞砸了？连按两次 `Esc` 即可回滚。但请注意，对数据库、API 等外部系统的操作无法回滚，所以它执行这类命令前一定会问你。
2. **权限控制**：按 `Shift+Tab` 可在四种模式间切换：默认（每次都问）、自动接受编辑、计划模式（只读不写，只出方案）、全自动模式（后台安全检查，实验性功能）。你也可以在配置中放行特定可信命令（如 `npm test`）。

## 高效协作的实战心法

最后，分享几个让 Claude Code 发挥最大威力的心法：

**1. 对话比完美的 Prompt 更重要**
不需要憋大招，直接开干，随时纠正：

```text
Fix the login bug
```
（Claude 尝试后方向不对？）
```text
That's not quite right. The issue is in the session handling.
```

**2. 给它验证的标尺**
告诉它期望的输出，或者贴上 UI 截图，让它能自我检验：

```text
Implement validateEmail. Test cases: 'user@example.com' → true,
'invalid' → false, 'user@.com' → false. Run the tests after.
```

**3. 委派而非微操**
把它当成熟的同事，给方向给上下文，别教它怎么敲键盘：

```text
The checkout flow is broken for users with expired cards.
The relevant code is in src/payments/. Can you investigate and fix it?
```

**4. 先探索后实施**
遇到复杂问题，先用计划模式（`Shift+Tab` 两次）让它只调研不写码，出方案后再动手。
