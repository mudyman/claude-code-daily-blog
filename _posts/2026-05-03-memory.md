# 告别健忘症：用 CLAUDE.md 和 Auto Memory 打造 Claude Code 的超强记忆

你是否有过这样的体验：每次开启新的 Claude Code 会话，都要重新解释一遍项目的构建命令、代码规范和架构设计？AI 助手虽然聪明，但“金鱼记忆”一直是痛点。今天我们来深入探讨 Claude Code 的记忆系统，看看如何让它真正成为懂你项目的老搭档。

## 双剑合璧：两种记忆机制

Claude Code 拥有两套互补的记忆系统，每次对话开始时都会加载：

- **CLAUDE.md 文件**：由你编写的持久化指令，相当于给 AI 的“员工手册”
- **Auto memory（自动记忆）**：Claude 根据你的纠正和偏好自动记录的笔记，相当于 AI 自己的“小本本”

| 维度 | CLAUDE.md | Auto memory |
| :--- | :--- | :--- |
| **谁来写** | 你 | Claude |
| **内容** | 指令和规则 | 学习到的模式和偏好 |
| **作用域** | 项目、个人或组织 | 单个 Git 仓库 |
| **加载量** | 全量加载 | 前 200 行或 25KB |

简单来说：你想规范 AI 的行为，用 CLAUDE.md；你想让 AI 自己长记性，靠 Auto memory。

## CLAUDE.md：你的项目宪法

CLAUDE.md 是纯文本 Markdown 文件，Claude 在每次会话启动时都会读取。什么时候需要写它？当你发现自己**第二次**向 AI 解释同一件事时，就是添加规则的绝佳时机。

### 放在哪里最合适？

CLAUDE.md 可以放在不同位置，作用域各不相同，更具体的路径优先级更高：

| 作用域 | 位置 | 典型场景 |
| :--- | :--- | :--- |
| **组织级** | `/Library/Application Support/ClaudeCode/CLAUDE.md` (macOS) 等 | 公司安全策略、合规要求 |
| **项目级** | `./CLAUDE.md` 或 `./.claude/CLAUDE.md` | 项目架构、构建命令（提交到 Git） |
| **用户级** | `~/.claude/CLAUDE.md` | 个人代码风格偏好（所有项目生效） |
| **本地级** | `./CLAUDE.local.md` | 个人测试 URL（加入 `.gitignore`） |



![Claude Md Directory Structure](assets/images/screenshots/claude-md-directory-structure.png)



**懒人福音**：在项目中直接运行 `/init`，Claude 会自动分析代码库并生成一份初始的 CLAUDE.md！如果文件已存在，它还会建议改进项，而不是直接覆盖。

### 写出高服从性指令的秘诀

CLAUDE.md 是作为上下文注入的，而非强制配置。指令写得越具体，Claude 遵循得越好。

- **控制体积**：目标在 200 行以内，太长会消耗上下文窗口并降低服从性。
- **结构清晰**：用 Markdown 标题和列表分组，AI 也像人一样更爱看条理清晰的内容。
- **极度具体**：写 `使用 2 空格缩进` 而不是 `格式化代码`；写 `提交前运行 npm test` 而不是 `测试你的改动`。

### 引用外部文件与 AGENTS.md 兼容

如果项目已经有其他 AI 使用的 `AGENTS.md`，不用复制一份，直接在 CLAUDE.md 中引用即可：

```markdown
@AGENTS.md

## Claude Code

对 `src/billing/` 目录的修改请使用 plan mode。
```

你还可以用 `@` 语法引入 README 或其他文档：

```text
参见 @README 了解项目概况，@package.json 了解可用的 npm 命令。

# 额外指令
- git 工作流 @docs/git-instructions.md
```

## 进阶玩法：模块化规则与自动记忆

### 用 `.claude/rules/` 实现按需加载

大项目如果把所有规则塞进一个文件，上下文窗口会爆炸。在 `.claude/rules/` 目录下创建按主题分类的 `.md` 文件，甚至可以通过 YAML frontmatter 实现**路径匹配**：

```markdown
---
paths:
  - "src/api/**/*.ts"
---

# API 开发规则

- 所有 API 端点必须包含输入验证
- 使用标准错误响应格式
```

这样，只有当 Claude 读取 `src/api/` 下的文件时，这些规则才会被加载，极大地节省了上下文空间。



![Path Specific Rules Example](assets/images/screenshots/path-specific-rules-example.png)



### Auto Memory：让 AI 自己长记性

Auto memory 默认开启（需 v2.1.59+）。当你在对话中纠正 Claude（比如“别用 npm，用 pnpm”），它会自动将这些偏好记录到 `~/.claude/projects/<project>/memory/` 目录下。

- **加载机制**：每次会话只加载 `MEMORY.md` 的前 200 行或 25KB。详细笔记会被 Claude 自动移到 `debugging.md` 等主题文件中，按需读取。
- **审计与修改**：在会话中输入 `/memory`，你可以浏览、编辑或删除所有记忆文件。它们只是普通的 Markdown 文件。



![Memory Command Ui](assets/images/screenshots/memory-command-ui.png)



### 排错指南：为什么 Claude 不听我的？

1. 运行 `/memory` 检查文件是否被正确加载。
2. 检查指令是否太模糊，或有冲突的规则。
3. 注意：`/compact` 压缩上下文后，只有项目根目录的 CLAUDE.md 会被重新注入，子目录的规则需要再次访问相关文件才会加载。

## 结语

善用 CLAUDE.md 和 Auto Memory，就是给 Claude Code 注入项目灵魂。从手动编写规范到 AI 自动积累经验，这套记忆系统能让你的 AI 助手越用越顺手，彻底告别每次开聊都要“重新自我介绍”的尴尬。
