# Day 1: Meet Claude Code — The AI Assistant That Actually Knows Your Codebase

Ever spent an hour writing tests for a module you barely understand? Or tracing a bug across twelve files, only to realize the root cause was a typo in a config file three directories deep? What if you could just *describe* what you want and have an assistant that already understands your entire project do the heavy lifting?

That's exactly what Claude Code promises — and after digging through the official docs, I can tell you it's way more than just another AI chatbot bolted onto a terminal.

## What Is Claude Code?

Claude Code is an **agentic coding tool**. That's a critical distinction. It doesn't just suggest code snippets — it reads your codebase, edits files, runs commands, and integrates with your development tools. It's the difference between asking someone for advice and hiring someone who actually does the work.

Think of it this way: traditional AI coding assistants are like having a consultant who looks over your shoulder and makes suggestions. Claude Code is like having a teammate who can open files, write code, run tests, fix failures, commit changes, and open pull requests — all while understanding the context of your entire project.

## Where Can You Use It?

One of the most impressive things about Claude Code is its surface area. It's not locked into one environment. The same underlying engine powers all of these:

- **Terminal CLI** — The full-featured command-line experience
- **VS Code & Cursor** — Inline diffs, @-mentions, plan review right in your editor
- **JetBrains IDEs** — IntelliJ, PyCharm, WebStorm, and more
- **Desktop App** — A standalone app with visual diff review and scheduled tasks
- **Web Browser** — No local setup required, works on claude.ai/code

Your `CLAUDE.md` files, settings, and MCP servers work across *all* of them. Start a task at your desk, continue it from your phone. That's not a future promise — that's how it works today.

## Hands-on: Getting It Installed

Let's get practical. Here's how to install the CLI, which is where most developers will start.

**macOS, Linux, or WSL:**

```bash
curl -fsSL https://claude.ai/install.sh | bash
```

**Windows PowerShell:**

```powershell
irm https://claude.ai/install.ps1 | iex
```

**Windows CMD:**

```batch
curl -fsSL https://claude.ai/install.cmd -o install.cmd && install.cmd && del install.cmd
```



![Cli Install Terminal](assets/images/screenshots/cli-install-terminal.png)



A quick Windows troubleshooting note: if you see `The token '&&' is not a valid statement separator`, you're in PowerShell, not CMD. If you see `'irm' is not recognized`, you're in CMD, not PowerShell. Look at your prompt — `PS C:\` means PowerShell, plain `C:\` means CMD. Native Windows setups also require [Git for Windows](https://git-scm.com/downloads/win) installed first.

**Prefer a package manager?** You've got options:

```bash
brew install --cask claude-code
```

Homebrew offers two casks: `claude-code` tracks the stable channel (about a week behind, skips regressions) while `claude-code@latest` gets new versions immediately. Note that Homebrew installs don't auto-update — run `brew upgrade claude-code` manually.

On Windows, there's also:

```powershell
winget install Anthropic.ClaudeCode
```

WinGet installations also don't auto-update, so run `winget upgrade Anthropic.ClaudeCode` periodically.

Once installed, just navigate to your project and launch:

```bash
cd your-project
claude
```



![Claude First Launch](assets/images/screenshots/claude-first-launch.png)



You'll be prompted to log in on first use. That's it — you're running.

## What Can You Actually Do With It?

This is where things get exciting. The docs outline a *lot* of capabilities:

**Automate the tedious stuff:**

```bash
claude "write tests for the auth module, run them, and fix any failures"
```

This single command can write tests, execute them, see what fails, and iterate on fixes. No copy-pasting error messages back and forth.

**Git workflows:**

```bash
claude "commit my changes with a descriptive message"
```

Claude Code stages changes, writes meaningful commit messages, creates branches, and opens PRs. In CI, you can automate code review and issue triage with GitHub Actions or GitLab CI/CD.

**Unix composability** — This is the part that made me genuinely excited:

```bash
# Analyze recent log output
tail -200 app.log | claude -p "Slack me if you see any anomalies"

# Automate translations in CI
claude -p "translate new strings into French and raise a PR for review"

# Bulk operations across files
git diff main --name-only | claude -p "review these changed files for security issues"
```

Piping data *into* Claude Code? Chaining it with standard Unix tools? This is the kind of composability that makes it feel like a real power tool, not a toy.

**Beyond the CLI**, there's a deep well of capabilities:
- **MCP (Model Context Protocol)** — Connect to Google Drive, Jira, Slack, or custom tooling
- **CLAUDE.md** — Persistent instructions that Claude reads every session (coding standards, architecture decisions, preferred libraries)
- **Custom commands** — Package repeatable workflows like `/review-pr` or `/deploy-staging`
- **Hooks** — Run shell commands before/after Claude actions (auto-format after edits, lint before commits)
- **Multi-agent orchestration** — Spawn multiple agents working on different parts of a task simultaneously
- **Scheduled tasks** — Morning PR reviews, overnight CI analysis, weekly dependency audits

## Pro Tips

1. **Start with CLAUDE.md.** This is your project's "onboarding document" for Claude. Put your coding standards, architecture decisions, and preferred libraries here. It pays dividends immediately.

2. **Use the `-p` flag for scripting.** The `-p` flag runs Claude in non-interactive "pipe" mode, making it perfect for CI pipelines and shell scripts.

3. **Sessions are portable.** Start in your terminal, hand off to the Desktop app with `/desktop` for visual diff review, or pull a web session into your terminal with `claude --teleport`. You're not locked into one surface.

4. **Native Windows installs auto-update.** The direct install script keeps you current automatically. Homebrew and WinGet don't — remember to upgrade manually.

5. **Check the integrations table.** Claude Code connects to Slack, Chrome, Telegram, Discord, GitHub, GitLab, and more. Before you build something custom, check if there's already an integration.
