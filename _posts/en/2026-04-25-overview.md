# Day 1: Meet Claude Code — The AI Assistant That Actually Lives in Your Codebase

Ever spent an afternoon writing tests for a module you built three months ago? Or tracing a bug through twelve files across two services? What if you could just say *"write tests for the auth module, run them, and fix any failures"* — and watch it actually happen?

That's not a hypothetical. That's Claude Code.

## What Is Claude Code?

Claude Code is an **agentic coding tool** that doesn't just suggest snippets — it reads your entire codebase, edits files across multiple directories, runs commands, and integrates with your existing development tools. It's not a chatbot that happens to know code. It's a coding partner that works *inside* your project.

The key word is **agentic**. Claude Code doesn't wait for you to copy-paste context. It explores your codebase, makes a plan, executes across files, and verifies the result. It works in your terminal, your IDE, a desktop app, and your browser — all connected to the same engine.

## Where It Runs: Pick Your Surface

Claude Code meets you wherever you work. Every surface connects to the same underlying engine, so your `CLAUDE.md` files, settings, and MCP servers carry over.

### Terminal CLI

The full-featured experience. Install it, `cd` into any project, and go:

**macOS, Linux, WSL:**

```bash
curl -fsSL https://claude.ai/install.sh | bash
```

**Windows PowerShell:**

```powershell
irm https://claude.ai/install.ps1 | iex
```

**Windows CMD:**

```batch
curl -fsSL https://claude.ai/install.cmd -o install.cmd && install.cmd && del install.cmd
```



![Cli Install Terminal](/assets/images/screenshots/cli-install-terminal.png)



Prefer a package manager? You've got options:

```bash
brew install --cask claude-code
```

Or on Windows:

```powershell
winget install Anthropic.ClaudeCode
```

> **Note:** Homebrew and WinGet installations don't auto-update. Run `brew upgrade claude-code` or `winget upgrade Anthropic.ClaudeCode` periodically. Native installations on Windows/macOS update automatically in the background.

Then just navigate to your project and start:

```bash
cd your-project
claude
```

You'll be prompted to log in on first use. That's it.

### VS Code & Cursor

Inline diffs, @-mentions, plan review, and conversation history — right in your editor. Search for "Claude Code" in the Extensions view (`Cmd+Shift+X`), install it, then open the Command Palette and select **Open in New Tab**.

### Desktop App

A standalone app for visual diff review, multiple side-by-side sessions, scheduled tasks, and cloud sessions. Available for [macOS](https://claude.ai/api/desktop/darwin/universal/dmg/latest/redirect?utm_source=claude_code&utm_medium=docs) (Intel and Apple Silicon) and [Windows](https://claude.ai/api/desktop/win32/x64/setup/latest/redirect?utm_source=claude_code&utm_medium=docs).

### Web & Mobile

Zero local setup. Kick off long-running tasks and check back later, work on repos you don't have locally, or run tasks in parallel. Start at [claude.ai/code](https://claude.ai/code).

### JetBrains

A plugin for IntelliJ IDEA, PyCharm, WebStorm, and others. Install the [Claude Code plugin](https://plugins.jetbrains.com/plugin/27310-claude-code-beta-) from the JetBrains Marketplace.

## What You Can Actually Do With It

This is where it gets fun. Here's what Claude Code handles in real development workflows:

**Automate the tedious stuff.** Tests, lint fixes, merge conflicts, dependency updates, release notes:

```bash
claude "write tests for the auth module, run them, and fix any failures"
```

**Build features and fix bugs.** Describe what you want in plain language. Paste an error message. Claude Code traces through your codebase, identifies root causes, and implements fixes across multiple files.

**Git workflows, automated:**

```bash
claude "commit my changes with a descriptive message"
```

It stages changes, writes commit messages, creates branches, and opens PRs. In CI, automate code review and issue triage with GitHub Actions or GitLab CI/CD.

**Connect external tools via MCP.** The Model Context Protocol lets Claude Code read Google Drive docs, update Jira tickets, pull Slack data, or use your custom tooling.

**Teach it your project's conventions.** `CLAUDE.md` is a markdown file in your project root that Claude reads every session. Set coding standards, architecture decisions, preferred libraries. Claude also builds *auto memory* — saving learnings like build commands and debugging insights across sessions without you writing anything.

**Compose with Unix pipes.** Claude Code follows the Unix philosophy:

```bash
# Analyze recent log output
tail -200 app.log | claude -p "Slack me if you see any anomalies"

# Automate translations in CI
claude -p "translate new strings into French and raise a PR for review"

# Bulk operations across files
git diff main --name-only | claude -p "review these changed files for security issues"
```

**Run on a schedule.** Morning PR reviews, overnight CI failure analysis, weekly dependency audits — all automated via Routines (Anthropic-managed infrastructure) or Desktop scheduled tasks.

**Move sessions between surfaces.** Start in your terminal, continue on your phone with Remote Control, hand off to the Desktop app with `/desktop` for visual diff review, or pull a web session into your terminal with `claude --teleport`.

## The Integration Map

Here's a quick reference for routing tasks to the right surface:

| I want to... | Best option |
|---|---|
| Continue a session from my phone | Remote Control |
| Push events from Telegram/Discord/webhooks | Channels |
| Automate PR reviews and issue triage | GitHub Actions or GitLab CI/CD |
| Route bug reports from Slack to PRs | Slack integration |
| Debug live web applications | Chrome extension |
| Build custom agent workflows | Agent SDK |

## Pro Tips

- **Native Windows requires [Git for Windows](https://git-scm.com/downloads/win).** Install it first. WSL setups don't need it.
- **Confused by Windows shell errors?** If you see `The token '&&' is not a valid statement separator`, you're in PowerShell, not CMD. Your prompt shows `PS C:\` in PowerShell and just `C:\` in CMD.
- **Homebrew has two casks:** `claude-code` tracks stable (about a week behind, skips regressions), while `claude-code@latest` gets new versions immediately.
- **Most surfaces require a Claude subscription or Anthropic Console account.** The Terminal CLI and VS Code also support third-party providers.
