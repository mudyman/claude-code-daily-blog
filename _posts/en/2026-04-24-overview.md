# Day 1: Meet Claude Code — Your New AI Pair Programmer

Ever spent an afternoon writing tests for code you wrote three months ago? Or traced a bug through fifteen files only to find a missing semicolon? What if you could just *describe* what you want and watch it happen?

That's the promise of Claude Code — and today, we're kicking off a deep-dive series into everything it can do.

## Introduction

Claude Code is an **agentic coding tool** that doesn't just autocomplete your brackets. It reads your entire codebase, edits files across multiple directories, runs commands, and integrates with your existing development tools. The key word here is *agentic* — it doesn't wait for you to spoon-feed it context. It explores, plans, and executes.

What makes it different from a chatbot with a terminal? Claude Code understands the *relationships* in your code. It can trace a bug from an error message through your call stack, write tests that actually cover the right edge cases, and stage + commit the results — all from a single prompt.

## Key Concepts

Before we install anything, let's anchor on the core ideas:

- **Codebase awareness**: Claude Code reads your project structure, dependencies, and conventions — not just the file you have open.
- **Multi-surface**: The same engine runs in your terminal, VS Code, JetBrains IDEs, a desktop app, and your browser. Your settings and memory travel with you.
- **Composable**: It follows Unix philosophy. Pipe logs into it, chain it with other tools, run it in CI — it plays well with others.
- **Extensible via MCP**: The Model Context Protocol connects Claude Code to external data sources like Google Drive, Jira, Slack, or your own custom tooling.
- **Persistent memory**: `CLAUDE.md` files and auto-memory mean Claude Code learns your project's conventions across sessions.

## Hands-on Walkthrough: Getting Claude Code Running

Let's get it installed. The primary interface is the **Terminal CLI**, and installation is straightforward.

### macOS, Linux, or WSL

```bash
curl -fsSL https://claude.ai/install.sh | bash
```

### Windows PowerShell

```powershell
irm https://claude.ai/install.ps1 | iex
```

### Windows CMD

```batch
curl -fsSL https://claude.ai/install.cmd -o install.cmd && install.cmd && del install.cmd
```

> **Stuck on Windows?** If you see `The token '&&' is not a valid statement separator`, you're in PowerShell, not CMD. If you see `'irm' is not recognized`, you're in CMD, not PowerShell. Look at your prompt: `PS C:\` means PowerShell, plain `C:\` means CMD. Also, native Windows setups require [Git for Windows](https://git-scm.com/downloads/win) — install that first.



![Cli Install Terminal](/assets/images/screenshots/cli-install-terminal.png)



### Alternative Install Methods

If you prefer package managers:

**Homebrew (macOS):**
```bash
brew install --cask claude-code
```
Note: `claude-code` tracks the stable channel (about a week behind), while `claude-code@latest` gets new versions immediately. Homebrew installations don't auto-update, so run `brew upgrade` periodically.

**WinGet (Windows):**
```powershell
winget install Anthropic.ClaudeCode
```

Native installations (via the curl method) auto-update in the background. Nice quality-of-life touch.

### Your First Session

Navigate to any project and launch:

```bash
cd your-project
claude
```

You'll be prompted to log in on first use. That's it — you're in.



![First Session Launch](/assets/images/screenshots/first-session-launch.png)



### Other Surfaces

The CLI is just the beginning. Claude Code also runs in:

- **VS Code / Cursor**: Search "Claude Code" in Extensions (`Cmd+Shift+X`), then open via Command Palette → "Open in New Tab"
- **Desktop App**: Download from claude.ai for [macOS](https://claude.ai/api/desktop/darwin/universal/dmg/latest/redirect) or [Windows](https://claude.ai/api/desktop/win32/x64/setup/latest/redirect) — includes visual diff review and scheduled tasks
- **Web**: Head to [claude.ai/code](https://claude.ai/code) for zero-setup coding in your browser
- **JetBrains**: Install the [Claude Code plugin](https://plugins.jetbrains.com/plugin/27310-claude-code-beta-) from the Marketplace

All surfaces share the same engine, so your `CLAUDE.md` files, settings, and MCP servers work everywhere.

## Pro Tips

**Start with the tedious stuff.** Claude Code shines brightest on tasks you've been putting off:

```bash
claude "write tests for the auth module, run them, and fix any failures"
```

**Pipe real data into it.** This is where composability gets powerful:

```bash
# Analyze recent log output
tail -200 app.log | claude -p "Slack me if you see any anomalies"

# Automate translations in CI
claude -p "translate new strings into French and raise a PR for review"

# Bulk operations across files
git diff main --name-only | claude -p "review these changed files for security issues"
```

**Use it for git workflows.** Don't just code — commit:

```bash
claude "commit my changes with a descriptive message"
```

**Think in surfaces.** Start a long-running task on the web, then pull it into your terminal with `claude --teleport`. Hand off a terminal session to the Desktop app with `/desktop` for visual diff review. Mention `@Claude` in Slack with a bug report and get a PR back. The workflow is fluid.

**Set up CLAUDE.md early.** This markdown file in your project root gets read at the start of every session. Put your coding standards, architecture decisions, and preferred libraries there. Claude Code also builds *auto memory* — saving learnings like build commands and debugging insights across sessions without you writing anything.
