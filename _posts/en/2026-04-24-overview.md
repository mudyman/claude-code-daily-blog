# Claude Code: The AI Coding Assistant That Lives Where You Work

Ever spent an afternoon writing tests for code you wrote three months ago? Or tracing a bug through fifteen files across two services? What if you could just *say* "write tests for the auth module, run them, and fix any failures" — and watch it happen?

That's the promise of Claude Code, Anthropic's agentic coding tool. And today, we're kicking off a deep-dive series that will take you from zero to power user.

## What Is Claude Code?

Claude Code isn't just another autocomplete plugin. It's an **agentic** coding assistant — meaning it doesn't just suggest the next line, it actually *does things*. It reads your entire codebase, edits files across multiple directories, runs commands, and integrates with your existing development tools.

Think of it as a deeply capable pair programmer who never gets tired, never forgets the architecture doc, and can spin up three parallel agents to tackle different parts of a refactor simultaneously.

The key insight: Claude Code meets you *where you already work*. Terminal, VS Code, JetBrains, desktop app, browser — it runs everywhere, and your settings, memories, and configurations follow you across all of them.

## Key Concepts

Before we install anything, let's understand what makes Claude Code tick:

- **Agentic, not just assistive** — Claude Code doesn't wait for you to copy-paste context. It explores your codebase, makes a plan, executes across files, and verifies the result.
- **Surface-agnostic** — The same engine powers the CLI, IDE extensions, desktop app, and web. Start a task in your terminal, continue on your phone, review diffs on the desktop.
- **Composable** — It follows Unix philosophy. Pipe logs into it, chain it with other tools, run it in CI. It's a first-class citizen in your toolchain.
- **Extensible** — Through MCP (Model Context Protocol), custom commands, hooks, and the Agent SDK, you can connect it to anything from Jira to Slack to your own internal tooling.

## Hands-on Walkthrough: Getting Claude Code Installed

Let's get it running. The primary interface is the CLI, and installation is straightforward.

### Terminal CLI Installation

**macOS, Linux, and WSL:**

```bash
curl -fsSL https://claude.ai/install.sh | bash
```

**Windows PowerShell:**

```powershell
irm https://claude.ai/install.ps1 | iex
```

**Windows CMD:**

```batch
curl -fsSL https://claude.ai/install.cmd -o install.cmd && install.cmd && del install.cmd
```

> **Windows users, heads up:** If you see `The token '&&' is not a valid statement separator`, you're in PowerShell, not CMD. If you see `'irm' is not recognized`, you're in CMD, not PowerShell. Your prompt shows `PS C:\` in PowerShell and just `C:\` in CMD. Also, native Windows setups require [Git for Windows](https://git-scm.com/downloads/win) — install that first. WSL users don't need it.

Native installations auto-update in the background, so you'll always be on the latest version.

### Alternative Install Methods

If you prefer package managers:

**Homebrew (macOS):**

```bash
brew install --cask claude-code
```

Note: `claude-code` tracks the stable channel (about a week behind), while `claude-code@latest` gets new versions immediately. Homebrew installs don't auto-update, so run `brew upgrade` periodically.

**WinGet (Windows):**

```powershell
winget install Anthropic.ClaudeCode
```

Same caveat — no auto-updates. Run `winget upgrade Anthropic.ClaudeCode` when you want the latest.

You can also install via **apt, dnf, or apk** on Debian, Fedora, RHEL, and Alpine.

### Your First Session

Once installed, navigate to any project and launch:

```bash
cd your-project
claude
```

### Other Surfaces

The CLI is just the beginning. Claude Code is available in:

- **VS Code / Cursor** — Search "Claude Code" in Extensions (`Cmd+Shift+X`), then open via Command Palette → "Claude Code: Open in New Tab"
- **Desktop App** — Download for [macOS](https://claude.ai/api/desktop/darwin/universal/dmg/latest/redirect?utm_source=claude_code&utm_medium=docs) or [Windows](https://claude.ai/api/desktop/win32/x64/setup/latest/redirect?utm_source=claude_code&utm_medium=docs). Visual diff review, multiple sessions, scheduled tasks.
- **Web** — Head to [claude.ai/code](https://claude.ai/code) for zero-setup coding in your browser. Great for long-running tasks or repos you don't have locally.
- **JetBrains** — Install the [Claude Code plugin](https://plugins.jetbrains.com/plugin/27310-claude-code-beta-) from the Marketplace.

Most surfaces require a [Claude subscription](https://claude.com/pricing?utm_source=claude_code&utm_medium=docs&utm_content=overview_pricing) or [Anthropic Console](https://console.anthropic.com/) account. The Terminal CLI and VS Code also support third-party providers.

## Pro Tips

**Pipe everything.** Claude Code is a Unix citizen. Some patterns that will change your workflow:

```bash
# Analyze recent log output
tail -200 app.log | claude -p "Slack me if you see any anomalies"

# Automate translations in CI
claude -p "translate new strings into French and raise a PR for review"

# Bulk operations across files
git diff main --name-only | claude -p "review these changed files for security issues"
```

**Use CLAUDE.md for persistent context.** Drop a markdown file in your project root with coding standards, architecture decisions, and preferred libraries. Claude reads it at the start of every session. It also builds *auto memory* — saving learnings like build commands and debugging insights across sessions without you writing anything.

**Git is a first-class citizen.** Don't just ask Claude to write code — ask it to commit, branch, and open PRs:

```bash
claude "commit my changes with a descriptive message"
```

**Go parallel with sub-agents.** Spawn multiple Claude Code agents working on different parts of a task simultaneously, with a lead agent coordinating and merging results.

**Schedule the boring stuff.** Morning PR reviews, overnight CI failure analysis, weekly dependency audits — Claude Code can run on a schedule via Routines (Anthropic-managed, runs even when your computer's off) or Desktop scheduled tasks.
