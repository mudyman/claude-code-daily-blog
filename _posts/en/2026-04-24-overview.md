# Day 1: Meet Claude Code — The AI Assistant That Actually Knows Your Codebase

Ever pasted an error message into ChatGPT, gotten a generic answer, and thought *"yeah, but you don't know my code"*? That's the gap Claude Code was built to fill. It's not just another chatbot — it's an agentic coding tool that lives inside your project, reads your entire codebase, edits files, runs commands, and integrates with your existing dev tools. Today, we're kicking off a deep-dive series into everything Claude Code can do.

---

## What Is Claude Code?

Claude Code is an AI-powered coding assistant that operates with full context of your project. Unlike browser-based chat, it can:

- **Read your codebase** and understand how files relate to each other
- **Edit files directly** across multiple files in a single operation
- **Run commands** — tests, builds, linters, whatever you need
- **Integrate with your tools** — git, CI/CD, Slack, Jira, and more

It's available across five surfaces: Terminal CLI, VS Code (and Cursor), a standalone Desktop app, your browser, and JetBrains IDEs. Same engine, same settings, same `CLAUDE.md` files — just different ways to interact depending on where you are.

## Getting It Installed

Let's get practical. The fastest way to start is the terminal CLI.

**macOS, Linux, WSL:**

```bash
curl -fsSL https://claude.ai/install.sh | bash
```

**Windows PowerShell:**

```powershell
irm https://claude.ai/install.ps1 | iex
```

**Windows CMD:**

```batch
curl -fsSL https://claude.ai/install.cmd -o install.cmd && install.cmd && del install.cmd
```



![Cli Install Terminal](/assets/images/screenshots/cli-install-terminal.png)



A few notes from the trenches: if you see `The token '&&' is not a valid statement separator`, you're in PowerShell, not CMD. If you see `'irm' is not recognized`, you're in CMD, not PowerShell. Look at your prompt — `PS C:\` means PowerShell, bare `C:\` means CMD. Native Windows setups also require [Git for Windows](https://git-scm.com/downloads/win). WSL users don't need it.

**Prefer a package manager?**

```bash
brew install --cask claude-code
```

The `claude-code` cask tracks the stable channel (about a week behind, skips regressions). Want bleeding edge? Use `claude-code@latest`. Note: Homebrew installs don't auto-update, so run `brew upgrade` periodically.

On Windows, there's also:

```powershell
winget install Anthropic.ClaudeCode
```

Same deal — no auto-updates, so run `winget upgrade` from time to time.

Once installed, just navigate to your project and launch:

```bash
cd your-project
claude
```

You'll authenticate on first use, and you're in.



![Claude First Launch](/assets/images/screenshots/claude-first-launch.png)



**Other surfaces:** You can also install the [VS Code extension](vscode:extension/anthropic.claude-code) (search "Claude Code" in Extensions), download the [Desktop app](https://claude.ai/api/desktop/darwin/universal/dmg/latest/redirect?utm_source=claude_code&utm_medium=docs) for macOS/Windows, code directly at [claude.ai/code](https://claude.ai/code) in your browser, or grab the [JetBrains plugin](https://plugins.jetbrains.com/plugin/27310-claude-code-beta-) from the Marketplace.

## What Can You Actually Do With It?

This is where things get exciting. Claude Code isn't just "answer questions about code" — it's "do the work."

### Automate the Tedious Stuff

```bash
claude "write tests for the auth module, run them, and fix any failures"
```

That one-liner handles writing tests, executing them, and iterating on failures. No more procrastinating on test coverage.

### Build Features from Plain Language

Describe what you want. Claude Code plans the approach, writes code across multiple files, and verifies it works. For bugs, paste an error message or describe the symptom — it traces through your codebase to find the root cause and implements a fix.

### Git Workflows

```bash
claude "commit my changes with a descriptive message"
```

It stages changes, writes meaningful commit messages, creates branches, and opens PRs. In CI, you can automate code review and issue triage with GitHub Actions or GitLab CI/CD.

### MCP: Connect External Tools

The Model Context Protocol (MCP) is an open standard for connecting AI to external data. Claude Code can read design docs in Google Drive, update Jira tickets, pull context from Slack, or use your own custom tooling.

### Persistent Memory with CLAUDE.md

Drop a `CLAUDE.md` file in your project root with coding standards, architecture decisions, and preferred libraries. Claude reads it at the start of every session. It also builds *auto memory* — saving learnings like build commands and debugging insights across sessions without you writing anything.

### Multi-Agent Workflows

Spawn multiple Claude Code agents working on different parts of a task simultaneously, with a lead agent coordinating and merging results. For full customization, the Agent SDK lets you build your own agents with control over orchestration, tool access, and permissions.

### Unix Philosophy: Pipe All The Things

```bash
# Analyze recent log output
tail -200 app.log | claude -p "Slack me if you see any anomalies"

# Automate translations in CI
claude -p "translate new strings into French and raise a PR for review"

# Bulk operations across files
git diff main --name-only | claude -p "review these changed files for security issues"
```

This composability is what separates a true CLI tool from a glorified chat window.

### Scheduled and Recurring Tasks

Run Claude on a schedule — morning PR reviews, overnight CI failure analysis, weekly dependency audits. Routines run on Anthropic-managed infrastructure (works even when your computer's off), while Desktop scheduled tasks run locally with access to your files.

### Seamless Session Portability

Start in the terminal, continue on your phone. Kick off a long-running task on the web, pull it into your terminal with `claude --teleport`. Hand off to the Desktop app with `/desktop`. Mention `@Claude` in Slack with a bug report and get a PR back.

## Choosing a Permission Mode

When you start using Claude Code, you'll need to decide how much autonomy it has. The permission mode controls whether Claude can execute commands and edit files without asking you first. This is critical for both safety and workflow speed — we'll cover the specifics in an upcoming post, but know that you have granular control over what Claude can do autonomously versus what requires your approval.

## Pro Tips

- **Start with the terminal** — it's the most flexible surface and gives you the full feature set, including piping and scripting
- **Create a `CLAUDE.md` immediately** — even a simple one with your project's tech stack and coding conventions dramatically improves output quality
- **Use the `-p` flag for scripting** — `claude -p "your prompt"` runs in non-interactive mode, perfect for CI pipelines and shell scripts
- **Native Windows installs auto-update** — Homebrew and WinGet don't, so set a reminder to upgrade manually
- **Sessions are portable** — don't feel locked into one surface. Start anywhere, continue everywhere
